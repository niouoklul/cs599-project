# Product Spec

## 1. 项目定位

项目名称：企业项目协同系统的智能运营 Agent 改造  
课程方向：方向二，企业级应用软件的 Agent 改造

系统面向项目型企业的项目经理、财务专员、实施工程师和系统管理员。改造前，用户需要在客户、项目、合同、工单、审批等模块之间手动检索、汇总和判断。改造后，用户可以通过自然语言向智能运营 Agent 提出目标，由 Agent 自动选择工具并执行多步骤任务。

## 2. 原始系统痛点

| 痛点 | 改造前表现 | Agent 改造目标 |
|---|---|---|
| 数据分散 | 风险信息分布在项目、工单、合同、审批模块 | 自动汇聚多表数据并输出风险原因 |
| 流程依赖人工 | 工单派单、合同审批需要人工判断 | 用工具调用完成派单和审批动作 |
| 管理报告成本高 | 周报需要人工复制指标和待办 | 一键生成经营周报 |
| 知识难复用 | 审批规则、派单规则只能靠经验 | 用知识库检索增强 Agent 答复 |
| 缺少可观测性 | 很难解释智能系统为何这样做 | 保存 Agent run 和 step 轨迹 |

## 3. 用户角色

| 角色 | 权限与目标 |
|---|---|
| 系统管理员 | 管理用户、查看审计日志、演示全部 Agent 能力 |
| 项目经理 | 管理客户和项目，审批合同，查看风险 |
| 财务专员 | 管理合同和审批，关注回款状态 |
| 实施工程师 | 处理工单，提交风险和问题 |

## 4. 核心用户故事

1. 作为项目经理，我希望输入“分析当前高风险项目”，系统自动汇总工单、合同和项目风险，输出风险原因。
2. 作为管理者，我希望输入“生成本周经营周报”，系统自动生成包含指标、风险、待办的周报。
3. 作为实施负责人，我希望输入“把 4 号紧急工单自动派单”，系统按规则选择处理人并更新工单。
4. 作为财务专员，我希望输入“通过合同 HT-2026-003”，系统完成合同审批并留下审计记录。
5. 作为评审者，我希望查看工具调用轨迹，确认 Agent 行为可解释。

## 5. 非功能要求

- 安全：密码 PBKDF2 哈希保存，API Key 只能来自环境变量。
- 权限：不同角色访问不同业务模块和工具动作。
- 可观测：保存 Agent 运行记录、工具调用参数、观察结果和审计日志。
- 可运行：默认不依赖外部 LLM，支持本地 deterministic planner，答辩时可离线演示。
- 可扩展：已支持免费 DeepSeek 兼容 API Tool Calling，并可继续演进到 LangGraph 或本地模型。

## 6. 验收标准

| 编号 | 验收项 |
|---|---|
| P1 | 用户可登录并访问业务模块 |
| P2 | Agent 可生成经营周报 |
| P3 | Agent 可分析项目风险并给出原因 |
| P4 | Agent 可调用工具修改工单或合同状态 |
| P5 | 系统保存 Agent 运行轨迹和审计日志 |
| P6 | 测试脚本与 benchmark 可运行 |
